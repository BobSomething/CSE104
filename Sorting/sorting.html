This is Bubble Sort: How it works?
  • It is comparing the current element with the one right next to it, swapping their values if they are not inorder. This operation is repeated until the array is sorted.
  • Worst-case performance: \(O(n^2)\)
  • Best-case performance: \(O(n)\)
  • Average performance: \(O(n^2)\)
  • To find out more details: Bubble Sort
This is Selection Sort: How it works?
  • It divides the list into two sublist: left - sorted and right - unsorted. Then it proceed to find the smallest element in the unsorted sublist and exchanging it with the first element of the unsorted sublist.
  • Worst-case performance: \(O(n^2)\)
  • Best-case performance: \(O(n^2)\)
  • Average performance: \(O(n^2)\)
  • To find out more details: Selection Sort
This is Merge Sort: How it works?
  • It is based on the concept of divide-and-conquer. It first recursively divides the list into two halves. And then repeatedly merging sublists to produce a sorted list.
  • Worst-case performance: \(O(n\log n)\)
  • Best-case performance: \(\Omega(n\log n)\)
  • Average performance: \(\Theta(n \log n)\)
  • To find out more details: Merge Sort
This is Quick Sort: How it works?
  • It is based on the concept of divide-and-conquer. It picks a pivot element from the list and dividing the other elements into two sublists, according to whether they are less or greater than the pivot.
  • Worst-case performance: \(O(n^2)\)
  • Best-case performance: \(O(n\log n)\)
  • Average performance: \(O(n \log n)\)
  • To find out more details: Quick Sort
This is Cocktail Sort: How it works?
  • It is an extension of Bubble Sort, it improves on operating in both directions, thus it moves more quickly elements to the beginning of the list.
  • Worst-case performance: \(O(n^2)\)
  • Best-case performance: \(O(n)\)
  • Average performance: \(O(n^2)\)
  • To find out more details: Cocktail Sort
This is Shell Sort: How it works?
  • It starts with sorting pairs of elements far apart from each other, then progressively reducing the gap between elements to be compared.
  • Worst-case performance: \(O(n^2)\)
  • Best-case performance: \(O(n\log n)\)
  • Average performance: Depends on the gap sequence.
  • To find out more details: Shell Sort
This is Pancake Sort: How it works?
  • It is based on the Pancake Problem. It iterate from the length of the list to 0, finds the minimum of the sublist to the current length considered. And flips sublists to the minimum element.
  • Worst-case performance: \(O(n^2)\)
  • Best-case performance: \(O(n^2)\)
  • Average performance: \(0(n^2)\)
  • To find out more details: Pancake Sort
This is Gnome Sort: How it works?
  • It works by building a sorted list one element at a time, getting each item to the proper place in a series of swaps.
  • Worst-case performance: \(O(n^2)\)
  • Best-case performance: \(O(n)\)
  • Average performance: \(O(n^2)\)
  • To find out more details: Gnome Sort
This is Brick Sort: How it works?
  • It is related to Bubble Sort. It functions by comparing all odd/even indexed pairs of adjacent elements in the list and, if a pair is in the wrong order the elements are switched.
  • Worst-case performance: \(O(n^2)\)
  • Best-case performance: \(O(n)\)
  • Average performance: \(O(n^2)\)
  • To find out more details: Brick Sort
This is Comb Sort: How it works?
  • It is improved version of Bubble Sort. It compares elements that have a gap bigger than 1, and gradually the initial gap shrinks.
  • Worst-case performance: \(O(n^2)\)
  • Best-case performance: \(\Omega(n\log n)\)
  • Average performance: \(\Theta(n^2/2^p)\), where \(p\) is the number of increments
  • To find out more details: Comb Sort
This is Stooge Sort: How it works?
  • It swaps the first and the last element if they are not inorder. Then it tries to sort the initial \(2/3\) of the list, then it sorts the final \(2/3\) of the list and again sorts the initial \(2/3\) of the list.
  • Worst-case performance: \(O(n^{\log 3 / \log 1.5})\)
  • To find out more details: Stooge Sort
This is Bogo Sort: How it works?
  • It shuffles the lists, and check if the lists is inorder. If it is, then the program terminates. If not, then it tries again shuffling, and so on.
  • Worst-case performance: \(O(?)\) - unbounded, with deterministic version \(O(n\cdot n!)\)
  • Best-case performance: \(\Omega(n)\)
  • Average performance: \(\Theta(n \cdot n!)\)
  • To find out more details: Bogo Sort
